## 1. Executive Summary

### 한 문장

**"Myverse는 나의 Personal Blackbox다. 사진 찍으면 AI가 정리하고, 소셜에 한 번에 공유하고, 서비스가 죽어도 내 기록은 남는다."**

### 사업 개요

Myverse는 개인의 일상을 기록하고 보관하는 모바일 앱이다. 스마트폰에 흩어진 수만 장의 사진, 여러 소셜에 분산된 기록, 서비스 종료와 함께 사라지는 디지털 흔적 — 이 문제를 해결한다.

사용자가 사진을 찍으면 AI가 자동으로 분류하고, 캘린더와 대조해 언제·어디서·누구와 찍었는지를 태깅한다. 인스타그램, 페이스북, X에 한 번에 공유할 수 있고, 공유 기록이 남는다. 기존 소셜에 쌓인 기록을 Myverse로 백업받을 수 있다. 10년, 20년 쌓이면 나의 인생이 타임라인으로 시각화되고, 자손에게 물려줄 수 있다.

Myverse는 Ten:One™ Universe의 빅뱅이다. 마케팅·광고 업계 전문가 네트워크에서 일반 대중을 위한 서비스로 확장하는 전환점이다.

## 2. 문제 정의

### 2-1. 흩어진 디지털 흔적

스마트폰에 수천, 수만 장의 사진이 있다. 누군가는 구글 포토를 쓰고, 누군가는 아이폰 사진을 쓴다. 인스타그램에 올리고, 페이스북에 공유하고, 스레드에 쓰고, X에 남긴다. 카카오톡으로 친구에게 보내고, 친구로부터 받는다.

여기저기 흩어져 있다. 일일이 관리하기도 힘들다. 어디에 뭘 올렸는지도 기억이 안 난다.

### 2-2. 사라지는 기록

그러다 서비스가 죽으면? 나의 흔적은?

- 2019년 싸이월드 — 수백만 명의 사진, 일기, BGM이 증발
- 2016년 바인(Vine) — 6초 영상 수십억 개가 사라짐
- 2019년 구글플러스 — 구글이 만든 소셜조차 폐쇄
- 2023년 클럽하우스 — 한때 전 세계를 뒤흔든 음성 소셜이 쇠퇴

내 디지털 인생인데, 소유권은 플랫폼에 있다. 플랫폼이 닫으면 내 기록도 끝이다.

### 2-3. 정리 안 되는 사진

스마트폰 갤러리에 사진이 수만 장 쌓여 있다. 정리할 엄두가 안 난다. 누구와 찍었는지, 어디서 찍었는지, 왜 찍었는지 — 시간이 지나면 맥락을 잊어버린다. 구글 포토가 얼굴 인식을 하지만, "이 사진이 내 인생에서 어떤 의미였는지"는 알려주지 않는다.

### 2-4. 소셜의 피로

인스타그램은 남에게 보여주는 공간이다. 꾸미게 된다. 좋아요 수에 신경 쓰게 된다. "나를 위한 기록"이 아니라 "남에게 보여주기 위한 전시"가 된다. 진짜 나의 일상을 솔직하게 기록할 공간이 없다.

## 3. 솔루션 — Myverse

### 3-1. Personal Blackbox

비행기에 블랙박스가 있는 이유는 무슨 일이 있어도 기록이 살아남아야 하기 때문이다.

Myverse는 나의 인생 블랙박스다.

- 모든 기록의 원본이 여기에 있다
- 소셜 플랫폼은 여기서 내보내는 채널일 뿐이다
- 서비스가 사라져도 내 기록은 Myverse에 남는다
- 개인 ID로 로그인한다. 본인이 아니면 열리지 않는다
- AI가 문지기 역할을 한다. 누군가 접속하려 하면 AI가 먼저 확인한다

### 3-2. 앱 구조 — 7탭

```
Myverse (iOS + Android 앱)
│
├── 👤 ME     — 나는 누구인가 (홈)
├── 📝 LOG    — 나의 일상 (사진·글·감정·창작)
├── 📅 PLAN   — 내 하루 (캘린더·할 일)
├── 🎯 DREAM  — 내가 꿈꾸는 것 (인생 목표)
├── 💼 WORK   — 내가 하는 일 (프로젝트)
├── 🤖 AI     — 나를 아는 AI (대화·분석)
└── 🌌 VERSE  — 내 우주 (인생 타임라인)
```

### 3-3. 핵심 기능

**① 사진을 찍으면 AI가 정리한다**

스마트폰에서 사진을 찍으면, Myverse가 카메라에서 직접 받는다. 앱 내 카메라로 촬영하거나, 기기 갤러리에서 사진을 선택하면 Myverse LOG에 올라간다.

AI가 캘린더 스케줄과 대조해서 자동 태깅한다.

```
📷 사진 촬영 또는 갤러리에서 선택
  ↓
AI 자동 분류:
  📍 어디서: 서울숲 카페 (위치 데이터 + 캘린더)
  📅 언제: 2026.03.27 오후 2시
  👤 누구와: 수진 (캘린더에 "수진이랑 점심" 일정)
  🏷️ 카테고리: 일상 > 식사 > 카페
  ↓
LOG에 자동 정리. 검색 가능.
```

기기에 이미 다운로드된 사진, 카카오톡에서 받은 사진, 스크린샷 — 갤러리에 있는 모든 파일을 Myverse로 가져올 수 있다. AI가 시간순으로 정리하고 맥락을 붙여준다.

**② 소셜에 한 번에 공유한다**

LOG에 쌓인 기록 중 공유하고 싶은 것을 선택하면, 인스타그램·페이스북·스레드·X에 동시에 공유한다. AI가 각 플랫폼에 맞는 캡션을 써준다.

```
📷 노을 사진 선택
  ↓
AI: "인스타 캡션: '퇴근길이 선물이 될 때도 있다.'
     X: '서울 노을.'
     인스타 + 페이스북 + X 동시 공유할까요?"
  ↓
[공유 완료]
  ├─ 인스타 ✓ (3/27 19:30)
  ├─ 페이스북 ✓ (3/27 19:30)
  └─ X ✓ (3/27 19:31)
  
공유 기록이 남는다. 어디에 뭘 올렸는지 추적 가능.
```

**③ 기존 기록을 끌어모아준다**

지금 시작해도 늦지 않다. 과거의 기록을 끌어모아준다.

```
가져오기 가능한 소스:
  📱 스마트폰 갤러리 (아이폰 / 안드로이드)
  ☁️ 구글 포토
  ☁️ 구글 드라이브
  📷 인스타그램 (데이터 다운로드 → 가져오기)
  📘 페이스북 (데이터 다운로드 → 가져오기)
  🐦 X / 트위터 (데이터 다운로드 → 가져오기)
  🎬 유튜브 (영상 목록 + 메타데이터)
  📝 블로그 (RSS 또는 데이터 내보내기)
```

각 소셜 플랫폼은 "내 데이터 다운로드" 기능을 제공한다(GDPR/개인정보보호법 의무). 사용자가 다운로드 받은 파일을 Myverse에 업로드하면, AI가 시간순으로 정리하고 자동 분류해서 나의 타임라인을 재구성한다.

**④ 기록하지 않아도 기록된다**

캘린더에서 오늘 뭘 했는지, 어디를 갔는지(위치), 누구와 소통했는지(이메일) — 자동으로 수집된다. 사용자가 아무것도 안 해도 하루가 기록된다. 직접 기록은 "더 넣고 싶을 때"만.

**⑤ 10년 후, 내 인생이 보인다**

VERSE 탭에서 쌓인 기록이 시각화된다.

- 타임라인 (자동 생성된 인생 연대기)
- 키워드 변화 (시기별 관심사 추이)
- 관계 변화 (누구와 가장 많이 만났는지)
- 성장 그래프 (목표 달성률)
- "1년 전 오늘 나는 이걸 했다"

**⑥ 나의 기록을 물려줄 수 있다**

디지털 유산 기능. 수신자를 지정하고, 공개 범위와 시점을 설정한다. 엄마의 인생을, 아빠의 일상을, Myverse로 물려준다.

## 4. 시장 분석

### 4-1. 시장 규모

**개인 클라우드 스토리지 시장:** 전 세계 $115B (2025) → $376B (2030) 예상. 연평균 성장률 27%.

**디지털 일기/라이프로그 앱 시장:** 전 세계 $1.2B (2025). Day One, Journey, Momento 등이 선두. 하지만 "AI 자동 분류 + 소셜 연동 + 블랙박스 보관"을 결합한 서비스는 없다.

**소셜 미디어 관리 도구:** Buffer, Hootsuite 등 크리에이터용 멀티 포스팅 도구가 $4.5B 시장. Myverse는 크리에이터뿐 아니라 일반 사용자까지 커버한다.

### 4-2. 경쟁 분석

| | 구글 포토 | 인스타그램 | Day One | Notion | **Myverse** |

### 4-3. 타깃 사용자

**Primary — 일반 대중**

스마트폰으로 사진 찍고, 인스타에 올리고, 카카오톡으로 공유하는 모든 사람. 20~40대 중심.

| 페르소나 | 설명 | 핵심 니즈 |
|---------|------|----------|
| 직장인 민지 (28) | 인스타에 일상을 올리지만 피곤함 | 나를 위한 기록. 남에게 보여주기가 아닌 |
| 워킹맘 수진 (35) | 아이 사진이 수만 장. 정리 불가 | AI가 정리해주고, 성장 기록으로 남기고 싶음 |
| 대학생 태호 (22) | 여러 소셜에 분산. 포트폴리오 필요 | 한 곳에서 관리하고, 커리어용으로 정리 |
| 은퇴자 영수 (58) | 인생 기록을 남기고 싶음 | 자손에게 물려줄 수 있는 디지털 유산 |

**Secondary — 크리에이터**

여러 소셜에 콘텐츠를 동시 배포하는 사람. Myverse에서 한 번 쓰고 멀티 플랫폼 공유.

**Tertiary — 조직 (Orbi 연동)**

MADLeague 학생, 기업 직원. 조직의 Orbi에 참여하면서 자연스럽게 Myverse 사용.

## 5. 제품 상세

### 5-1. 로그인 및 보안

**개인 ID 로그인:**
- 이메일 + 비밀번호 (기본)
- Google 계정 로그인
- Apple 계정 로그인
- 카카오 계정 로그인 (한국 시장)

**추가 보안 (선택):**
- Face ID / 지문 인식 — 원하는 사용자만 설정
- 2단계 인증 (OTP)
- 앱 잠금 (PIN 코드)

**AI 문지기:**
- 외부에서 내 데이터에 접근 시도 시 AI가 알림
- Orbi 연동 시 공유 범위 확인 요청
- 의심스러운 접근 자동 차단

**데이터 보안:**
- 종단간 암호화 (서버에서도 내 데이터를 읽을 수 없음)
- 데이터는 사용자 소유. WIO는 마케팅 목적으로 데이터를 사용하지 않음
- 계정 삭제 시 모든 데이터 완전 삭제

### 5-2. 카메라 및 파일 접근

**카메라 — 두 가지 경로:**

```
경로 1: 앱 내 카메라
  Myverse 앱에서 직접 카메라 실행
  → 촬영 → 바로 LOG에 저장 + AI 자동 분류
  → 가장 빠른 경로. 인스타 스토리 찍듯이.

경로 2: 기기 갤러리에서 선택
  스마트폰 기본 카메라로 촬영 후
  → Myverse 앱에서 갤러리 접근 → 사진 선택
  → LOG에 저장 + AI 자동 분류
  → 이미 찍어둔 사진을 올릴 때.
```

**파일 — 다운로드된 것들:**

```
스마트폰에 있는 파일:
  📷 갤러리 사진/영상         → Myverse가 직접 접근 (권한 허용 시)
  📁 다운로드 폴더 파일       → 파일 선택기로 가져오기
  💬 카카오톡에서 받은 사진   → 갤러리에 저장된 것은 접근 가능
  📎 이메일 첨부파일          → 파일 선택기로 가져오기

클라우드에 있는 파일:
  ☁️ 구글 포토               → Google Photos API 연동
  ☁️ 구글 드라이브           → Google Drive API 연동
  ☁️ 아이클라우드            → iOS 파일 접근 (제한적)

소셜에 있는 기록:
  📷 인스타그램              → "내 데이터 다운로드" 파일 업로드
  📘 페이스북                → "내 정보 다운로드" 파일 업로드
  🐦 X / 트위터             → "데이터 아카이브 요청" 파일 업로드
  🎬 유튜브                  → "데이터 다운로드" 파일 업로드
```

**핵심 원칙:**
- 스마트폰 갤러리는 사용자 권한 허용 시 직접 접근
- 클라우드 서비스는 API 연동 또는 파일 업로드
- 소셜 플랫폼은 각 서비스의 "내 데이터 다운로드" 기능을 활용
- AI가 어떤 경로로 들어온 파일이든 시간순 정리 + 맥락 분류

### 5-3. 7탭 상세

**ME (홈)** — 앱을 열면 보이는 첫 화면. 오늘의 기분, 최근 기록, 오늘 일정, 올해 목표. AI가 대화로 프로필을 끌어냄.

**LOG (일상)** — Myverse의 심장. 사진, 글, 감정, 창작, 일기. AI 자동 분류. 소셜 일괄 공유. 소셜 백업. 과거 기록 끌어모으기. 공유 기록 추적.

**PLAN (하루)** — 캘린더 동기화, 투두리스트, 타임박싱, 루틴 관리. AI가 "오늘 뭐 할래?" 매일 아침 브리핑.

**DREAM (꿈)** — 인생 비전, 연간 목표, 버킷리스트. AI가 질문으로 끌어내고 자동 분해.

**WORK (일)** — 프로젝트 관리, 프레임워크 라이브러리. Orbi 연동 시 조직 프로젝트가 여기에.

**AI (인공지능)** — 나를 가장 잘 아는 AI. 패턴 분석, 리뷰 자동 생성, 기획 가이드.

**VERSE (우주)** — 인생 타임라인, 키워드 변화, 관계 변화, 성장 그래프. 디지털 유산.

### 5-4. Orbi — 조직 연동

Myverse가 개인이라면, Orbi는 조직이다. 별도 웹 서비스.

조직(동아리, 회사, 프로젝트 팀)이 Orbi를 만들고, Myverse 사용자를 초대한다. 사용자는 자기 Myverse 안에서 조직 프로젝트에 참여한다. 개인 기록은 비공개, 업무만 공유.
