## 1. 핵심 철학

> "세상에 필요한 모든 기능을 모듈로 만든다. 필요에 따라 끼워 넣으면 된다."

WIO는 기업이 운영에 필요한 모든 것을 하나의 플랫폼에서 해결하는 시스템이다.
별도 백엔드, 별도 SaaS를 붙이지 않는다. WIO 모듈을 조합한다.

**핵심 포지셔닝**: "입력을 없앤다. AI가 80%를 채운다."

---

## 2. 3대 자원 프레임워크

모든 WIO 모듈은 아래 3대 자원 중 최소 하나를 관리한다.
모듈 간 연결도 이 3가지를 기준으로 흐른다.

| 자원 | 의미 | 모듈에서의 역할 |
|------|------|----------------|
| **사람 (Human)** | 누가, 몇 명, 어떤 역할 | People, Team, Permission |
| **돈 (Money)** | 얼마, 수익, 비용, 정산 | Finance, Budget, Billing |
| **시간 (Time)** | 언제까지, 몇 시간, 일정 | Timesheet, Schedule, Deadline |

예시:
- Project = 사람(투입 인원) + 돈(예산/수익) + 시간(기간/마감)
- Sales = 사람(담당자) + 돈(딜 가치) + 시간(클로즈 기한)
- Content = 사람(작성자) + 시간(발행 일정)

## 3. 7 Track 시스템

WIO의 모듈은 7개 트랙으로 분류된다. 트랙은 기업 기능 영역이다.

| Track | 이름 | 설명 | 주요 부서 |
|-------|------|------|----------|
| Track 1 | 운영 . 관리 | 경영전략, 인사, 재무, 법무 | 경영지원, HR, 재무, 법무 |
| Track 2 | 사업 | 마케팅, 영업, CRM, BD | 마케팅, 영업, 사업개발, 고객관리 |
| Track 3 | 생산 | 구매조달, 재고, 생산, 물류, 품질 | 생산, 구매, 물류, 품질 |
| Track 4 | 지원 | 개발, 디자인, R&D, 데이터, 콘텐츠 | 개발, 디자인, 연구, 데이터 |
| Track 5 | 파트너 | 파트너, 벤더, 프리랜서 | 파트너관리, 제휴 |
| Track 6 | 공통 (전 직원) | 협업, 커뮤니케이션, 전자결재 | 전사 |
| Track 7 | 시스템 관리 | 시스템 설정, 보안, 데이터 거버넌스 | IT, 보안 |

추가:
- **개인 (My)**: 전 직원 자동 제공 개인 대시보드
- **지주사 (Holding)**: TenOne 전용 브랜드/자회사 관리

## 7. 모듈 간 연결 구조

모듈은 독립적이되, 3대 자원(사람/돈/시간)을 기준으로 연결된다.

```
[프로젝트] --사람--> [인재/조직]
           --돈----> [예산/재무]
           --시간--> [시수/캘린더]

[영업]     --사람--> [고객관리]
           --돈----> [견적/매출]
           --시간--> [파이프라인 기한]

[채용]     --사람--> [인재]
           --돈----> [급여]
           --시간--> [온보딩 일정]
```

**연결 원칙:**
- 모듈 간 직접 JOIN 금지. API/이벤트로만 연동.
- 모든 테이블에 brand_id 컬럼 (멀티테넌트 RLS).
- API 응답은 APIResponse<T> 형식.

## 8. 워크플로우 3-Layer 설계

### 8.1 3-Layer 구조

| Layer | 이름 | 세팅 주체 | 범위 |
|-------|------|----------|------|
| Layer 1 | 전사 워크플로우 | 경영기획/시스템관리자 | 전사 공통 프로세스 |
| Layer 2 | 부서별 워크플로우 | 부서장 | 부서 고유 업무 프로세스 |
| Layer 3 | 업무 도구 | 부서장 + 자동 제공 | 부서 공통 도구 + 개인 업무 도구 |

### 8.2 Layer 1: 전사 워크플로우 (10종)

| # | 이름 | 트리거 | 관련 트랙 | SLA |
|---|------|--------|----------|-----|
| 1 | 전자결재 | 결재 요청 시 | 전 트랙 | 48시간 |
| 2 | 채용 | 채용 요청 시 | 운영관리 + 해당 부서 | 30일 |
| 3 | 온보딩 | 입사일 기준 | 운영관리 + 시스템 + 해당 부서 | 입사 후 7일 |
| 4 | 퇴직 | 퇴직 신청 시 | 운영관리 + 시스템 | 30일 전 |
| 5 | 구매 | 구매 요청 시 | 생산 + 운영관리 | 5영업일 |
| 6 | 예산 | 분기/연간 편성 | 전 트랙 + 운영관리 | 분기 1개월 전 |
| 7 | 평가 | 평가 기간 | 전 트랙 + 운영관리 | 2주 |
| 8 | 보상 | 평가 확정 후 | 운영관리 | 평가 후 1개월 |
| 9 | 신제품출시 | 출시 기획 승인 | 사업 + 생산 + 지원 | 프로젝트별 |
| 10 | 컴플라이언스 | 위반 감지 | 전 트랙 + 운영관리 | 즉시 |

### 8.3 Layer 2: 부서별 워크플로우

| 부서 | 워크플로우 | 단계 수 |
|------|-----------|---------|
| 마케팅 | 캠페인 실행, 콘텐츠 발행 | 4, 4 |
| 영업 | 딜 프로세스 | 4 |
| 개발 | 스프린트, 장애 대응 | 4, 4 |
| 디자인 | 디자인 요청 | 4 |
| 생산 | 생산 실행, 불량 처리 | 4, 4 |
| 인사 | 교육 운영 | 4 |
| 재무 | 월 마감 | 4 |

### 8.4 Layer 3: 업무 도구

**부서 공통 도구** (부서장이 모듈 카탈로그에서 선택 활성화):
- 해당 부서가 사용할 모듈을 선택하여 활성화
- 예: 마케팅 부서 -> mkt-strategy, mkt-campaign, MKT-ANL 등

**개인 업무 도구** (전 직원 자동 제공):
- MY-HOME, MY-HR, MY-EVL, MY-WRK, MY-APR
- 비활성화 불가

### 8.5 Layer 간 관계

```
Layer 1 (전사) : 모든 부서가 따라야 하는 표준 프로세스
    |
    v
Layer 2 (부서) : 부서 고유의 상세 워크플로우
    |
    v
Layer 3 (도구) : 실제 업무를 수행하는 모듈/도구
```

### 8.6 3-Layer 관계도

```
+-------------------------------------------------------+
|  Layer 1: 전사 워크플로우                                |
|  [전자결재] [채용] [온보딩] [퇴직] [구매]                   |
|  [예산] [평가] [보상] [신제품출시] [컴플라이언스]             |
|  세팅: 경영기획/시스템관리자                                |
+-------------------------------------------------------+
                         |
                         v
+-------------------------------------------------------+
|  Layer 2: 부서별 워크플로우                                |
|  마케팅: [캠페인실행] [콘텐츠발행]                           |
|  영업:   [딜프로세스]                                     |
|  개발:   [스프린트] [장애대응]                              |
|  생산:   [생산실행] [불량처리]                              |
|  세팅: 부서장                                            |
+-------------------------------------------------------+
                         |
                         v
+-------------------------------------------------------+
|  Layer 3: 업무 도구                                      |
|  [부서 공통 도구]         |  [개인 업무 도구]               |
|  부서장 선택 활성화       |  MY-HOME, MY-HR, MY-EVL,     |
|  모듈 카탈로그에서 선택    |  MY-WRK, MY-APR              |
|                          |  전 직원 자동 제공              |
+-------------------------------------------------------+
```

## 10. 모듈 개발 체크리스트

새 모듈을 만들 때 반드시 확인:

- [ ] 모든 테이블에 brand_id 컬럼이 있는가?
- [ ] RLS 정책이 brand_id 기반으로 적용됐는가?
- [ ] TenOne super_admin은 전체 접근 가능한가?
- [ ] 모듈 간 연동은 API/이벤트로만 하고 직접 JOIN은 없는가?
- [ ] API 응답 구조가 APIResponse<T> 형식을 따르는가?
- [ ] 이 모듈을 외부 기업이 써도 작동하는가?
- [ ] 7가지 입장(사용자/관리자/TenOne/외부/보안/확장/AI) 검증 완료?
- [ ] 3대 자원(사람/돈/시간) 중 최소 하나를 관리하는가?
- [ ] 해당 트랙에 올바르게 분류됐는가?
- [ ] wio-modules.ts MODULE_CATALOG에 등록됐는가?
