# WIO (Work In One) AI Business OS 개발 계획서

> Ten:One™ Universe 업무 운영 플랫폼
> "시스템이 알아서 일한다. 사람은 확인하고 판단만 한다."
> 최종 업데이트: 2026-03-26

---

## 1. WIO란

WIO는 Ten:One™ Universe의 **업무 운영 플랫폼(Business OS)**이다.

프로젝트 관리, CRM, HR, 회계, 지식베이스, 콘텐츠, 크롤링까지 — 8대 업무 영역을 하나로 묶되, **AI가 입력의 80%를 대신하고, 사람은 확인과 판단만 하는 구조**로 설계한다.

노션 템플릿과의 근본적 차이: 노션은 "사람이 넣고 AI가 정리". WIO는 **"AI가 넣고 사람이 확인"**.

### 핵심 원칙

```
입력을 없앤다. 입력이 불가피하면 AI가 80% 채워준다.
```

## 2. 8대 모듈 × AI 설계

### 2-1. WIO Project (프로젝트·업무 관리)

**기존 설계:** 프로젝트 등록, GPR 연동, Vrief 템플릿, 태스크 관리, 완료 트리거 체인
**AI 업데이트:**

| 기존 (수동) | AI 자동화 |
|------------|----------|
| 프로젝트 양식 직접 작성 | 카톡/슬랙/이메일로 "○○ 시작" → AI가 파싱 → GPR 템플릿 + 예산 + 일정 초안 자동 생성 → 확인만 |
| 태스크 수동 등록 | 회의록에서 액션 아이템 자동 추출 → 태스크 자동 생성 + 담당자 매칭 |
| 진행 상태 수동 업데이트 | Git 커밋, Slack 메시지, 캘린더에서 자동 추적 |
| 완료 시 수동 정산 | 완료 트리거 → 정산·포인트·아카이브·BI 동시 자동 업데이트 |
| 주간 리포트 직접 작성 | AI가 주간 활동 데이터 기반 리포트 초안 자동 생성 |

**외부 연동:**
- Slack/Discord → 메시지 기반 태스크 자동 생성
- Google Calendar → 일정·시수 자동 추출
- GitHub/GitLab → 코드 커밋 기반 진행률 자동 추적

## 3. AI 입력 3경로

```
경로 1: 완전 자동 (사람 개입 0%)
    └ TrendHunter 크롤링, 이메일/카톡 패턴 감지, 캘린더 동기화,
      Git 커밋 추적, 프로젝트 완료→정산 체인

경로 2: AI 초안 → 사람 확인 (AI 80% + 사람 20%)
    └ 프로젝트 등록, 회의록→액션 아이템, 시수표, 정산서,
      콘텐츠 초안, 주간 리포트, 인재 매칭 추천

경로 3: 간편 입력 (불가피할 때, 최소한으로)
    └ 카톡/슬랙 자연어 → AI 구조화
      사진(영수증/명함/계약서) → OCR + AI 파싱
      음성 메모 → 텍스트 변환 + 분류
```

## 4. 기술 스택

| 레이어 | 기술 | 비고 |
|--------|------|------|
| 프론트엔드 | Next.js 16 + React 19 + TypeScript + Tailwind CSS v4 | tenone.biz와 동일 |
| 백엔드 | Supabase (PostgreSQL + Auth + Edge Functions + Storage) | tenone.biz와 DB 공유 |
| AI 엔진 | Claude API (Sonnet 4.6 기본, Opus 4.6 복잡 태스크) | 초안 생성, 분류, 요약, 매칭 |
| OCR | Google Cloud Vision API 또는 Claude Vision | 영수증, 명함, 계약서 |
| 음성→텍스트 | OpenAI Whisper API | 음성 메모, 회의 녹음 |
| 크롤링 | TrendHunter 크롤러 (Node.js + Puppeteer + 메신저봇R) | DB 공유 |
| 배포 | Google Cloud Run | tenone.biz와 동일 인프라 |
| 실시간 | Supabase Realtime | 알림, 상태 변경 실시간 반영 |

## 6. 사람 개입이 필요한 영역

AI가 다 하는 게 아니다. 사람이 해야 할 것과 투자가 필요한 것을 명확히 구분.

### 6-1. 사람이 반드시 해야 할 것

| 역할 | 구체적 업무 | AI 대체 불가 이유 |
|------|-----------|-----------------|
| **의사결정** | 프로젝트 승인, 예산 확정, 파트너 선정 | 판단과 책임 |
| **관계 관리** | 클라이언트 미팅, 파트너 협상, 네트워킹 | 신뢰와 인간관계 |
| **품질 검수** | AI 산출물 최종 확인, 브랜드 톤 검수 | 감각과 경험 |
| **전략 방향** | 사업 방향, 신규 서비스 기획, 포지셔닝 | 비전과 철학 |
| **영업** | 신규 고객 개척, 제안, 클로징 | 설득과 관계 |

### 6-2. 투자가 필요한 영역

| 영역 | 투자 내용 | 시기 | 예상 비용 |
|------|----------|------|----------|
| **서버/인프라** | GCP Cloud Run + Supabase Pro | Phase 1 | 월 $50~100 |
| **API 비용** | Claude API + Google Vision + Whisper | Phase 2 | 월 $50~200 |
| **외부 API 계약** | 바로빌, 솔라피, 카카오 등 | Phase 3 | 월 10~50만원 |
| **개발 인력** | Claude Code + 텐원 (초기) → 주니어 개발자 1명 (안정화 후) | Phase 4 | 인건비 |
| **디자인** | UX/UI 고도화 (초기는 Claude가 작성, 고도화 시 디자이너 필요) | Phase 4 | 외주 또는 인턴 |
| **법률/회계** | 개인정보 처리방침, 이용약관, 사업자등록, 세무 | Phase 3 | 초기 100~300만원 |
