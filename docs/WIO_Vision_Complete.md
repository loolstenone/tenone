# Work In One (WIO)
## 기업과 사회가 원하는 솔루션

> Ten:One™ Universe 내부를 위해 만들지만
> 세상 모든 조직이 쓸 수 있게 설계한다

---

## 1. 왜 WIO가 필요한가 — 시장이 말하는 문제

### 지금 기업들이 겪는 현실

기업들은 평균 5~7개의 툴을 동시에 쓴다.
슬랙으로 소통하고, 노션에 문서 쓰고, 지라로 프로젝트 관리하고,
세일즈포스로 영업 관리하고, 엑셀로 재무 정리한다.

결과: 정보가 파편화된다. 툴 비용이 쌓인다. 사람은 툴에 맞춰 일한다.

세계 기업용 협업툴 시장은 2026년 858억 달러(약 113조원) 규모로 성장 중이다.
그런데 국내 기업의 90%는 여전히 카카오톡으로 업무 소통을 한다.
이 갭이 WIO의 시장이다.

### 기존 솔루션의 한계

| 솔루션 | 강점 | 한계 |
|--------|------|------|
| 노션 | 문서·위키 | 실시간 협업·알림 약함. 성과관리 없음 |
| 슬랙 | 실시간 소통 | 성과관리 없음. 정보 파편화 |
| 먼데이닷컴/아사나 | 프로젝트 관리 | 커뮤니티·교육·HR 없음 |
| 네이버웍스/카카오워크 | 한국어 최적화 | AI 기능 약함. 커뮤니티 없음 |
| 세일즈포스 | CRM | 중소기업 도입 어려움. 비쌈 |

**모든 기존 툴의 공통 한계**:
커뮤니티 + 교육 + 인재 + 비즈니스 + AI를 하나에서 제공하는 솔루션이 없다.

## 2. WIO가 만드는 세상 — 포지셔닝

### "Work In One"의 의미

하나의 플랫폼에서 일의 모든 것이 연결된다.

```
사람이 모인다 (커뮤니티)
    → 배운다 (교육)
    → 함께 일한다 (프로젝트)
    → 성과를 만든다 (비즈니스)
    → AI가 자동화한다
    → 다시 연결로 돌아온다
```

이게 WIO의 플라이휠이다.

### 경쟁사와의 차별점

```
노션 + 슬랙 + 세일즈포스 + 탈렌트LMS + 커뮤니티 플랫폼
= 월 수백만원 × 여러 팀이 각각 써야 하는 것들

WIO = 이걸 하나로. 한국어 최적화. AI 내장. 커뮤니티까지.
```

### 타깃

**Primary — 중소기업·스타트업·에이전시 (10~200명)**
- 다양한 툴을 쓰다 지친 곳
- 커뮤니티와 교육이 핵심 사업인 곳
- HR·재무·프로젝트를 통합하고 싶은 곳

**Secondary — 커뮤니티·협회·교육기관**
- 회원 관리 + 교육 + 이벤트 + 콘텐츠가 필요한 곳
- 기존 그룹웨어가 맞지 않는 곳

**Tertiary — 대기업 부서·팀 단위**
- 사내 스핀오프 프로젝트팀
- 사내 커뮤니티·동호회 운영

## 3. WIO의 핵심 가치 제안

### "입력을 없앤다. AI가 80%를 채운다."

기존 협업툴의 가장 큰 문제는 사람이 툴에 맞춰 일한다는 것이다.
WIO는 반대로 설계한다. 툴이 사람에게 맞춰진다.

```
회의 후 → AI가 회의록 자동 작성
프로젝트 시작 → Vrief 템플릿 자동 제안
시수 입력 → 캘린더 기반 자동 채워짐
트렌드 수집 → AI가 콘텐츠 초안 자동 생성
채용 요청 → AI가 후보 자동 매칭
```

### "연결이 일이다."

WIO 안에서 사람들이 연결되면 일이 생긴다.
일이 생기면 프로젝트가 만들어진다.
프로젝트가 끝나면 성과가 쌓인다.
성과가 쌓이면 더 좋은 연결이 생긴다.

이것이 WIO가 단순 협업툴이 아닌 이유다.

## 5. 설계 원칙 — 충돌 없이 확장하는 법

### 멀티테넌트 아키텍처

```
단일 Supabase DB + Row Level Security (RLS)

모든 테이블에 brand_id 컬럼
→ 자기 조직 데이터만 접근 가능
→ TenOne super_admin은 전체 접근
→ 조직 간 데이터 공유는 API/이벤트로만

Why: 스키마 분리하면 조직 간 연결이 불가능해진다
     WIO의 핵심은 연결이기 때문에 단일 DB가 맞다
```

### 모듈 독립성

```
각 모듈은 다른 모듈 없이도 작동한다
모듈 간 연동은 이벤트(webhook)나 API 호출로만

Why: 모듈을 독립적으로 업데이트·교체할 수 있어야
     WIO가 지속적으로 발전할 수 있다
```

### 역할 체계 통일

```
system_role: super_admin / brand_admin / member / guest
brand_role: 각 조직이 커스텀 (leaguer, crew, talent 등)

Why: Universe 23개 브랜드가 같은 역할 언어로 소통해야
     인재가 브랜드 간 이동할 때 마찰이 없다
```

## 6. 사회가 원하는 것 — WIO의 사회적 역할

### AI 시대의 새로운 고용 형태

AI가 신입이 하던 일을 대신한다.
기업은 더 적은 사람을 고용하고 더 많은 것을 자동화한다.
이 변화에서 소외되는 사람들이 생긴다.

WIO는 이 문제에 대한 답이다.

```
고용 형태가 아닌 연결 형태
계약서가 아닌 신뢰가 먼저
한 조직에 소속이 아닌 여러 프로젝트에 연결

WIO 안에서는 누구나
- 자신의 역량을 증명하고
- 프로젝트에 투입되고
- 성과를 쌓고
- 더 좋은 연결을 만들 수 있다
```

### 지역과 청년을 위한 플랫폼

수도권에 없어도 된다.
사무실에 나가지 않아도 된다.
AI를 도구로 쓸 줄 알면 된다.

WIO는 지역의 청년이 서울의 기업과 같은 프로젝트를 할 수 있는 환경을 만든다.

### "약한 연결로 강한 기회를"

강한 고용 관계가 아니어도 된다.
한 번 같이 일한 사람은 WIO 안에 남는다.
필요할 때 다시 연결된다.
그 네트워크가 커리어가 된다.

## 9. Claude Code에게 — 개발 체크리스트

```
새 모듈 만들 때마다 확인:

□ 모든 테이블에 brand_id 컬럼이 있는가?
□ RLS 정책이 brand_id 기반으로 적용됐는가?
□ TenOne super_admin은 brand_id 없이 전체 접근 가능한가?
□ 모듈 간 연동은 API/이벤트로만 하고 직접 JOIN은 없는가?
□ API 응답 구조가 APIResponse<T> 형식을 따르는가?
□ 테이블 네이밍이 [module]_[resource] 패턴인가?
□ 이 모듈을 TenOne Universe 외부 기업이 써도 작동하는가?

마지막 질문이 핵심이다.
WIO는 Ten:One™을 위해 만들지만
세상 모든 조직이 쓸 수 있게 설계한다.
```
