# WIO 개발 체크리스트 — 브랜드별 × 모듈별 × 입장별

> 복잡한 세계관인만큼 문제가 발생하면 안 된다.
> 모든 모듈을 만들 때 이 체크리스트를 통과해야 한다.

---

## 모듈 개발 시 반드시 확인할 7가지 입장

모듈 하나를 만들 때, **7가지 입장**에서 검증한다:

```
1. 사용자 입장    — 이걸 쓰는 사람이 불편하지 않은가?
2. 관리자 입장    — 이걸 관리하는 사람이 통제할 수 있는가?
3. TenOne 입장   — 23개 브랜드를 종합 관리할 수 있는가?
4. 외부 고객 입장 — Ten:One 밖의 기업이 써도 작동하는가?
5. 보안 입장     — 다른 브랜드 데이터가 보이면 안 된다
6. 확장 입장     — 10배 사용자가 와도 무너지지 않는가?
7. AI 입장       — AI가 자동화할 수 있는 부분은 어디인가?
```

## 크로스 브랜드 주의사항

### 한 사람이 여러 브랜드에 소속될 때

```
예: 김철수
├── TenOne (staff)
├── MADLeague (8기 리거)
├── Badak (현업 멤버)
└── SmarComm (프로젝트 크루)
```

**확인:**
- [ ] 로그인 한 번으로 모든 브랜드 접근 가능한가? (SSO)
- [ ] 브랜드 간 전환이 매끄러운가?
- [ ] 각 브랜드에서의 역할/권한이 독립적인가?
- [ ] 프로필은 하나인데 역할은 브랜드별로 다른가?
- [ ] 탈퇴 시 특정 브랜드만 탈퇴 가능한가?

### 데이터가 브랜드를 넘나들 때

```
예: MADLeague 우수 리거 → HeRo 인재 추천
```

**확인:**
- [ ] 사용자 동의 없이 데이터가 다른 브랜드로 넘어가면 안 됨
- [ ] "HeRo에 인재로 등록하시겠습니까?" 명시적 동의
- [ ] 동의 철회 시 연동 데이터 삭제
- [ ] 크로스 브랜드 데이터 이동은 API/이벤트로만 (직접 DB JOIN 금지)

### 브랜드별 디자인이 다를 때

```
예: MADLeague(빨강) vs Badak(다크블루) vs HeRo(오렌지)
```

**확인:**
- [ ] 같은 WIO 모듈이지만 브랜드 테마가 적용되는가?
- [ ] 로고, 악센트 컬러, 푸터가 브랜드별로 바뀌는가?
- [ ] "Powered by WIO" 또는 "Powered by Ten:One™" 표시

## 개발 순서 원칙

```
1. 공통 모듈부터 (Home → People → Talk → Content → Insight)
2. 한 모듈을 완성하면 23개 브랜드에 즉시 적용
3. 특화 기능은 해당 브랜드에서 먼저 만들고 범용화
4. 매 모듈마다 7가지 입장 체크리스트 통과
5. 크로스 브랜드 테스트 (최소 3개 브랜드에서 동시 테스트)
```
